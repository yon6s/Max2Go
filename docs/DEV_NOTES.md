# Max2Go 协作开发与修订日志

本文件用于多 Agent 之间的经验传承与信息同步。每次遇到复杂的业务逻辑改动、特殊的 Bug 修复、或者影响全局的架构约定，请务必以简短的条目记录在此。

## 全局架构与省流优化 (近期约定)
- **Token 优化机制**：前端传给后端 API 的表单数据已做了空字段过滤；在不需要竞品分析的模块（如合同、看板、测算），后端 `app/prompts.php` 已做了拦截，不再向大模型灌输长篇幅的 `project_knowledge`，以此大幅降低 API 成本。
- **计算逻辑纯代码化**：凡是能用确定性算法解决的（如 J回正插值计算、Word 合同变量替换），都已用 PHP/JS 直接实现，严禁在这些环节调用大模型。
